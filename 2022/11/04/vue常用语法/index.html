vue模板语法有2大类：

  1. 插值语法：用于解析标签体内容。写法：，xxx是js表达式，可以直接读取到data中的所有属性
  2. 指令语法：用于解析标签（标签属性、标签体内容、绑定事件……），指令形式都是v-xxx

1.v-bind（单向数据绑定）

动态绑定属性，比如图片的链接src ，网站的链接href，动态绑定一些类，样式等

写法：v-bind：属性名=“xxx” ， xxx同样要写成js表达式

“v-bind：”可以简写成“：”

2.v-model(双向数据绑定)

使用v-bind，vue实例中的data改变后，可以改变标签属性值，但是页面标签属性值改变不能导致实例中的data也改变。v-model则可以双向影响。

“v-model：value=…”可以简写成“：v-model=….”因为model默认收集的都是value值

  • v-model只能应用在表单类元素（输入类元素，有value值的）

3.v-show

条件渲染，当条件为false时，将元素的display属性设置为none。

4.v-if

v-if后面的条件为false时，对应的元素以及其子元素不会渲染，也就是根本不会有对应的标签出现在DOM中。

开发中如何选择呢？

  • 当需要在显示与隐藏之间切片很频繁时，使用v-show
  • 当只有一次切换时，通常使用v-if

5.v-for（key属性）

当有一组数据需要进行渲染时，就可以使用v-for来完成，语法类似于for循环

key属性：是虚拟DOM对象的标识

官方推荐我们在使用v-for时，给对应的元素或组件添加上一个:key属性。如果遍历时没有添加key属性，那么VUE默认以index作为key.

通过新旧节点中key值相同的元素，来对比。vue拿到数据之后都会生成虚拟DOM，然后利用key属性，使用对比算法比较新旧DOM组织。在对比过程中：右边新虚拟DOM首先看第一个li,key=’0’,然后查找初始虚拟DOM中key=’0’的li,比较所有节点（图中文本节点‘老刘-30’和元素节点’<input type ……>’）,对比发现文本节点内容不一致，于是使用新值生成DOM节点；元素节点一致，于是使用初始DOM节点。以此类推，依次生成DOM节点。

使用id作为key,在这种情况下不会错乱。

  1. 虚拟DOM中key的作用:
    key是虚拟DOM对象的标识，当数据发生变化时，Vue会根据【新数据】生成【新的虚拟DOM】,随后Vue进行【新虚拟DOM】与【旧虚拟DOM】的差异比较，比较规则如下:

  2. 对比规则:
    (1).旧虚拟DOM中找到了与新虚拟DOM相同的key:

         若虚拟DOM中内容没变,直接使用之前的真实DOM !
     若虚拟DOM中内容变了，则生成新的真实DOM，随后替换掉页面中之前的真实DOM。

    (2).旧虚拟DOM中未找到与新虚拟DOM相同的key

         创建新的真实DOM，随后渲染到页面。

  3. 用index作为key可能会引发的问题:

    1. 若对数据进行:逆序添加、逆序删除等破坏顺序操作:
      会产生没有必要的真实DOM更新==〉界面效果没问题,但效率低。
    2. 如果结构中还包含输入类的DOM:
      会产生错误DOM更新==>界面有问题。

  4. 开发中如何选择key? :
    1.最好使用每条数据的唯一标识作为key，比如id、手机号、身份证号、学号等唯一值。

    2.如果不存在对数据的逆序添加、逆序删除等破坏顺序操作，仅用于渲染列表用于展示,使用index作为key是没有问题的。

6.v-on

用于绑定事件。v-on后面的值是一个方法。

对应事件的方法定义在methods选项中。

7.v-test

将文本数据显示在界面中,但是不够灵活。可以使用插值语法

8.v-once

  • 该指令后面不需要跟任何表达式(比如之前的v-for后面是由跟表达式的)
  • 该指令表示元素和组件(组件后面才会学习)只渲染一次，不会随着数据的改变而改变。

9.v-html(xss攻击)

  • 某些情况下，我们从服务器请求到的数据本身就是一个HTML代码
    • 如果我们直接通过{{}}来输出，会将HTML代码也一起输出。 - 但是我们可能希望的是按照HTML格式进行解析，并且显示对应的内容。 - 如果我们希望解析出HTML展示,可以使用v-html指令，其会将string的html解析出来并进行渲染 >**cookie**:某些网站为了辨别用户身份，进行 Session 跟踪而储存在用户本地终端上的数据（通常经过加密），由用户客户端计算机暂时或永久保存的信息. 客户端向服务器发起请求，如果服务器需要记录该用户状态，就使用response向客户端浏览器颁发一个Cookie。客户端浏览器会把Cookie保存起来。当浏览器再请求该网站时，浏览器把请求的网址连同该Cookie一同提交给服务器。服务器检查该Cookie，以此来辨认用户状态。 当用户第一次访问并登陆一个网站的时候，cookie的设置以及发送会经历以下4个步骤： **客户端发送一个请求到服务器** --》 **服务器发送一个HttpResponse响应到客户端，其中包含Set-Cookie的头部** --》 **客户端保存cookie，之后向服务器发送请求时，HttpRequest请求中会包含一个Cookie的头部** --》**服务器返回响应数据** >站点上动态渲染的任意html可能会非常危险，他容易导致xss攻击。请只对可信内容使用html插值，绝不要对用户提供的内容使用插值 ### 10.v-cloak(页面闪现) v-cloak指令（没有值）: 1. 本质是一个特殊属性，Vue实例创建完毕并接管容器后，会删掉v-cloak属性。 2. 使用css配合v-cloak，规则如 `[v-cloak] { display: none }` 可以解决网速慢时页面展示出{xxx}}的问题（vue还未接管容器，纯html展示在页面上）。

11.v-pre

  1. 跳过其所在节点的编译过程。
  2. 可利用它跳过:没有使用指令语法、没有使用插值语法的节点，会加快编译。

vue.set()和vm.$set

当vue实例已经创建好了，有时候需要再次给数据赋值时，并不能在视图中改变。

Vue.set() 方法

参数2： 属性
参数3： 属性的值
返回值：已经修改好的值

检测数组更新

Vue是响应式的，所以当数据发生变化时，Vue会自动检测数据变化，视图会发生对应的更新

  • push()
  • pop()
  • shift() 移除数组的第一项。
  • unshift() 将新项添加到数组的开头，并返回新的长度
  • splice() 添加或删除数组中的元素。
  • sort() 进行排序。升序function(a, b){return a-b}，降序function(a, b){return b-a}
  • reverse()
