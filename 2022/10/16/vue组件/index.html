vue组件

组件是可复用的vue实例，实现应用中局部功能代码和资源的集合。代码复用率高，扩展性强。

一个组件的 data 选项必须是一个函数，因此每个实例可以维护一份被返回对象的独立的拷贝，如果仍然是个对象则所有实例将共享引用同一个数据对象。

组件的嵌套

VueComponent

  1. school组件本质是一个名为VueComponent的构造函数，且不是程序员定义的，是Vue.extend生成的。
  2. 我们只需要写或，Vue解析时会帮我们创建school组件的实例对象，即Vue帮我们执行的: new VueComponent(options).
  3. 特别注意:每次调用Vue.extend，返回的都是一个全新的VueComponent! ! ! !
  4. 关于this指向;
    (1).组件配置中:
    data函数、methods中的函数、watch中的函数、computed中的函数它们的this均【VueComponent实例对象】.
    (2).new Vue(options)配置中:
    data函数、methods中的函数、watch中的函数、computed中的函数它们的this均是【Vue实例对象】.
  5. VueComponent的实例对象，以后简称vc（也可称之为:组件实例对象）。
    Vue的实例对象，以后简称vm

有这个关系之后：让组件实例对象（vc）可以访问到Vue原型上的属性、方法。

关于不同版本的Vue:

  1. vue.js 与vue.runtime.xxx .js的区别:
    (1).vue.js是完整版的Vue，包含:核心功能+模板解析器。
    (2).vue.runtime.xxx.js是运行版的Vue，只包含:核心功能;没有模板解析器。
  2. 因为vue.runtime.xxx.js没有模板解析器，所以不能使用template配置项，需要使用render函数接收到的createElement函数去指定具体内容。

ref

  1. 被用来给元素或子组件注册引用信息(id的替代者)
  2. 应用在html标签上获取的是真实DOM元素。应用在组件标签上是组件实例对象(vc)
  3. 使用方式:
    打标识:<h1 ref=“xxx”>…..（标签上）或（组件上）
    获取: this.$refs.xxx

可以获取子组件的data；可以调用子组件中的方法

mixin

组件之间共有的配置可以抽离成一个混入对象。

混合中可以写：methods,name,data，，，，。当组件使用混入对象时，所有混入对象的选项将被“混合”进入该组件本身的选项。

当组件和混入对象含有同名选项时，以组件数据优先,而如果是生命周期钩子函数，则两边都要。另外，混入对象的钩子将在组件自身钩子之前调用。

混入也可以进行全局注册。使用时格外小心！一旦使用全局混入，它将影响每一个之后创建的 Vue 实例

scoped

用于在样式中限定作用域，防止不同的组件当中有相同的样式名，在使用的时候错乱。它的原理其实是在元素中添加了一个附加属性，其值是随机生成的，因此不会和其他同名的样式混淆

组件间通信的方式

props

组件实例的作用域是孤立的。这意味着不能 (也不应该) 在子组件的模板内直接引用父组件的数据。要让子组件使用父组件的数据，需要通过子组件的 props 选项。（vc的配置项，与template同级别。）父组件==》子组件

功能:让组件接收外部传过来的数据
(1).传递数据:

(2).接收数据:
第一种方式（只接收):
props : [ ‘name ‘] 然后子组件利用插值语法，使用这个数据
第二种方式(限制类型):
props:{
name : String
}
第三种方式(限制类型、限制必要性、指定默认值）:
props:{
name:{
type:String, //类型

​ required:true, //必要性

​ default:’老王’ //默认值
​ }
备注: props是只读的，Vue底层会监测你对props的修改，如果进行了修改，就会发出警告，若业务需求确实需要修改，那么请复制props的内容到data中一份，然后去修改data中的数据。

自定义事件

一种组件间通信的方式，适用于：子组件==》父组件。

A父组件，B子组件。B想给A传数据，就应该在A中给B绑定自定义事件。

绑定方式：

  1. 父组件中：<soncomponent @事件名=‘方法’>或<soncomponent v-on：事件名=‘方法’>。在方法中实现数据的接收
  2. 在父组件中：this.$refs.soncomponent.$on(‘事件名’，方法)

触发方式（子组件中）：

this.$emit(‘事件名’,数据)

解绑方式：

this.$off(‘事件名’)
