  • {{}}内可以写js表达式，如：1+1，Date.now() - 真实开发中只有一个vue实例，并且会配合组件一起使用 - **vue实例中如果有箭头函数，函数内的this不是指向vue实例，而会往上一层找** MVVM模型 ![MVVM](:vue:/mvvm.png) view: 视图层。也就是dom层，主要用于给用户展示各种信息\n model：数据层。来自于服务器中的数据\n vueModel: 视图模型层。是视图和数据的桥梁。一方面实现了数据绑定，将model的改变实时反映到view中，另一方面实现了DOM监听，当DOM发生一些事件时，可以监听到，并在需要的情况下改变对应的Data。 ### 工作原理 ![工作原理](:vue:/工作原理.png) 1. 通过建立虚拟dom树`document.createDocumentFragment()`,方法创建虚拟dom树。 2. 一旦被监测的数据改变，会通过[Object.defineProperty]定义的数据拦截，截取到数据的变化。 3. 截取到的数据变化，从而通过订阅——发布者模式，触发Watcher（观察者）,从而改变虚拟dom的中的具体数据。 4. 最后，通过更新虚拟dom的元素值，从而改变最后渲染dom树的值，完成双向绑定 ### 生命周期 开始创建、初始化数据、编译模板、挂载Dom、渲染→更新→渲染、卸载。 当一个 Vue 实例被创建时，它将 `data` 对象中的所有的 property 加入到 Vue 的**响应式系统**中。当这些 property 的值发生改变时，视图将会产生“响应”，即匹配更新为新的值。当这些数据改变时，视图会进行重渲染。 **生命周期函数中的this指向的是vm或者组件实例对象** ![生命周期钩子函数](:vue:/生命周期钩子函数.png) 上图涉及的函数都称为钩子函数，钩子就是在某个阶段给你一个做某些处理的机会。 **beforeCreate( 创建前 )** 数据监测和数据代理并未初始化，因此无法访问methods， data， computed等上的方法和数据 **created ( 创建后 ）** 实例已经创建完成，实例已完成以下配置：数据观测、属性和方法的运算，watch/event事件回调，完成了data 数据的初始化，el没有。 **beforeMount（挂载前）** 实例已完成以下的配置： 编译模板，把data里面的数据和模板生成html，完成了el和data 初始化，注意此时还没有挂在html到页面上。形成的是虚拟DOM **mounted（挂载后）** 挂载完成，也就是模板中的HTML渲染到HTML页面中，此时一般可以做一些ajax操作，启动定时器，绑定自定义事件、订阅消息等，mounted只会执行一次。 **beforeUpdate（更新前）** 在数据更新之前被调用，发生在虚拟DOM重新渲染和打补丁之前，可以在该钩子中进一步地更改状态，不会触发附加地重渲染过程 **updated（更新后）** 在由于数据更改导致地虚拟DOM重新渲染和打补丁时会调用，该钩子在服务器端渲染期间不被调用 **beforeDestroy（销毁前）** 实例仍然完全可用， 1. 这一步还可以用this来获取实例， 2. 一般在这一步做一些重置的操作，比如清除掉组件中的定时器 和 监听的dom事件 **destroyed（销毁后）** 在实例销毁之后调用，调用后，所有的事件监听器会被移出，所有的子实例也会被销毁，该钩子在服务器端渲染期间不被调用 ## 基础语法 vue模板语法有2大类： 1. 插值语法：用于解析标签体内容。写法：，xxx是js表达式，可以直接读取到data中的所有属性 2. 指令语法：用于解析标签（标签属性、标签体内容、绑定事件……），指令形式都是v-xxx ### 1.v-bind（单向数据绑定） 动态绑定属性，比如图片的链接src ，网站的链接href，动态绑定一些类，样式等 写法：v-bind：属性名=“xxx” ， xxx同样要写成js表达式 **“v-bind：”可以简写成“：”** ### 2.v-model(双向数据绑定) 使用v-bind，vue实例中的data改变后，可以改变标签属性值，但是页面标签属性值改变不能导致实例中的data也改变。v-model则可以双向影响。 **“v-model：value=...”可以简写成“：v-model=....”**因为model默认收集的都是value值 + v-model只能应用在表单类元素（输入类元素，有value值的） ### 3.v-show 条件渲染，当条件为false时，将元素的display属性设置为none。 ### 4.v-if v-if后面的条件为false时，对应的元素以及其子元素不会渲染，也就是根本不会有对应的标签出现在DOM中。 **开发中如何选择呢？** - 当需要在显示与隐藏之间切片很频繁时，使用v-show - 当只有一次切换时，通常使用v-if ### 5.v-for（key属性） 当有一组数据需要进行渲染时，就可以使用v-for来完成，语法类似于for循环 key属性：是虚拟DOM对象的标识 通过新旧节点中key值相同的元素，来对比。vue拿到数据之后都会生成虚拟DOM，然后利用key属性，使用对比算法比较新旧DOM组织。在对比过程中：右边新虚拟DOM首先看第一个li,key='0',然后查找初始虚拟DOM中key='0'的li,比较所有节点（图中文本节点‘老刘-30’和元素节点''）,对比发现文本节点内容不一致，于是使用新值生成DOM节点；元素节点一致，于是使用初始DOM节点。以此类推，依次生成DOM节点。 ![虚拟DOM2](:vue:/虚拟DOM2.png) 使用id作为key,在这种情况下不会错乱。 1. 虚拟DOM中key的作用: key是虚拟DOM对象的标识，当数据发生变化时，Vue会根据【新数据】生成【新的虚拟DOM】,随后Vue进行【新虚拟DOM】与【旧虚拟DOM】的差异比较，比较规则如下: 2. 对比规则: (1).旧虚拟DOM中找到了与新虚拟DOM相同的key: 若虚拟DOM中内容没变,直接使用之前的真实DOM ! 若虚拟DOM中内容变了，则生成新的真实DOM，随后替换掉页面中之前的真实DOM。 (2).旧虚拟DOM中未找到与新虚拟DOM相同的key 创建新的真实DOM，随后渲染到页面。 3. 用index作为key可能会引发的问题: 1. 若对数据进行:逆序添加、逆序删除等破坏顺序操作: 会产生没有必要的真实DOM更新==〉界面效果没问题,但效率低。 2. 如果结构中还包含输入类的DOM: 会产生错误DOM更新==>界面有问题。 4. 开发中如何选择key? : 1.最好使用每条数据的唯一标识作为key，比如id、手机号、身份证号、学号等唯一值。 2.如果不存在对数据的逆序添加、逆序删除等破坏顺序操作，仅用于渲染列表用于展示,使用index作为key是没有问题的。 ### 6.v-on 用于绑定事件。v-on后面的值是一个方法。 computed和watch之间的区别

  • computed能完成的功能，watch都可以完成。
  • watch能完成的功能，computed不一定能完成，例如: watch可以进行异步操作。

两个重要的小原则:

  1. 所被Vue管理的函数，最好写成普通函数，这样this的指向才是vm或组件实例对象。
  2. 所有不被Vue所管理的函数（定时器的回调函数、ajax的回调函数等、Promise的回调函数)，最好写成箭头函数，这样this的指向才是vm或组件实例对象。

组件

组件是可复用的vue实例，实现应用中局部功能代码和资源的集合。代码复用率高，扩展性强。

一个组件的 data 选项必须是一个函数，因此每个实例可以维护一份被返回对象的独立的拷贝，如果仍然是个对象则所有实例将共享引用同一个数据对象。

路由器工作模式（hash和history）

#及其后面的内容就是hash值。hash值不会带给服务器，不会包含在http请求中。

hash 模式（默认是此模式）
1.地址中永远带着#号，不美观
2.若以后将地址通过第三方手机app分享，若app校验严格，则地址会被标记为不合法
3.兼容性较好
4.由于hash符号之前的内容才会被包含在请求中，因此对于后端来说即使没有做到对路由的全覆盖也不会出现404错误。

history模式
1.地址干净，美观
2.兼容性和hash模式相比略差
3.应用部署上线时需要后端人员支持，解决刷新页面服务端404的问题
4.前端的 URL 必须和实际向后端发起请求的 URL 一致，否则返回404错误。

Router官网里如此描述:“不过这种模式要玩好，还需要后台配置支持…….所以呢，你要在服务端增加一个覆盖所有情况的候选资源:如果URL匹配不到任何静态资源，则应该返回同一个index.html 页面，这个页面就是你app依赖的页面。”

浏览器内核

  1. 渲染引擎:负责取得网页内容（html、xml、图像等）、处理讯息，然后输出至打印机
  2. JS引擎：解析JS语言，执行JS语言来实现网页的动态效果

（2）Gecko(firefox)。特点是代码完全公开，因此，其可开发程度很高

（3） webkit(Safari)。是苹果公司开发的浏览器，所用浏览器内核的名称是大名鼎鼎的WebKit

（4）Chromium/Bink(chrome)。内置于 Chrome 浏览器之中。Blink 其实是 WebKit 的分支

（5） Presto(Opera)。是挪威产浏览器 opera 的”前任” 内核。最新的opera 浏览器已经也实用Blink内核了

移动端内核：webkit、Bink、Trident、Gecko等。其中iphone、ipad等ios平台主要是webkit。Android 4.4 之前的 Android 系统浏览器内核是 WebKit，Android4.4 系统浏览器切换到了 Chromium，内核是Webkit 的分支 Blink，Windows Phone 8 系统浏览器内核是 Trident。

web标准

不同的浏览器解析出来的效果可能不一致，通过Web 标准展示统一内容。它不是某一个标准，而是由 W3C 和其他标准化组织制定的一系列标准的集合。主要包括结构〈Structure)、表现(Presentation）和行为〈Behavior）三个方面。

**结构标准(html):**结构用于对网页元素进行整理和分列，主要包括 XML 和 XHTML 两个部分。

**祥式标准(css):**表现用于设置网页元素的版式、颜色、大小等外观样式，主要指的是 CSS。

**行为标准(js):**行为是指网页模型的定义及交互的编写，主要包括 DOM 和 ECMAScript 两个部分。

注意∶

1.浮动的元素是互相贴靠在一起的(不会有缝隙），如果父级宽度装不下这些浮动的盒子，多出的盒子会另起一行对齐。

2.如果块级盒子没有设置宽度，默认宽度和父级一样宽，但是添加浮动之后，他的大小根据内容来定

3.一般先用标准流的父元素上下布局，再用浮动的子元素左右布局

4.浮动的盒子只会影响浮动盒子后面的标准流，不会影响前面的标准流

清除浮动

​ 清除浮动元素造成的影响

如果父元素没有给定高度，其内的子元素是浮动的，由于浮动元素不占用空间，那么此时父元素的高度将变为0.对于这种情况，需要清除浮动，清除浮动之后，父级就会根据浮动的子盒子自动检测高度，父级有了高度就不会影响下面的标准流。

定位

可以让盒子自由的在某个盒子内移动位置或者固定屏幕中某个位置，并且可以压住其他盒子

​ 定位=定位模式+边偏移

定位模式position属性

值 语义
static 静态定位（按照标准流摆放位置，没有边偏移）
relative 相对定位（移动的时候，以原来的位置为参照点，原来的位置还保留，后面盒子不会跑过去）
absolute 绝对定位（如果没有祖先元素或者祖先元素没定位，则以浏览器为准定位）
fixed 固定定位（固定于浏览器中某个位置，页面滚动时不会变化）

绝对定位（ absolute ）：如果没有祖先元素或者祖先元素没定位，则以浏览器为准定位；如果祖先有定位（相对、绝对、固定定位），则以最近一级的有定位祖先元素为参考点移动位置；绝对定位不再占有原先的位置; 绝对定位的盒子不能通过margin:0 auto 水平居中,可以通过计算宽度来确定位置

口诀:子绝父相 （这种定位使用的最多）

固定定位（fixed）:始终以可视窗口为参照点移动元素；与父元素没有关系；固定定位不占有原先的位置

固定定位,定位在版心右侧小技巧：

让固定定位的盒子left:50%,走到可视区的一半位置；再让盒子margin-left:版心宽度的一半距离。

边偏移属性 示例 描述
top top:80px; 元素相对于其父元素上边线的距离
bottom bottom:80px; 元素相对于其父元素下边线的距离
left left:80px; 元素相对于其父元素左边线的距离
right right:80px; 元素相对于其父元素右边线的距离

粘性定位（sticky）:可以被认为是相对定位和固定定位的混合。语法：选择器{position：sticky;top:10px;} 以浏览器的可视窗口为参照点移动元素，粘性定位占有原先的位置，必须添加top,left,right,bottom其中一个才有效。

定位模式 是否脱标 移动位置 是否常用
static定位 否 不能使用边偏移 很少
relative定位 否（占有位置） 相对于自身位置移动 常用
absolute定位 是（不占有位置） 带有定位的父级 常用
fiexd定位 是（不占有位置） 浏览器可视区 常用
sticky粘性定位 否（占有位置） 浏览器可视区 当前阶段少

数值越大，盒子越靠上，只有定位的盒子才有这个属性

语法格式

flex布局

也叫弹性布局。设置此布局的容器存在主轴和侧轴，默认子元素都沿主轴排列。

父项常见属性

设置此布局的容器有以下属性可以使用：

  • flex-direction 设置主轴的方向
  • flex-wrap 主轴排不下时，决定是否换行显示
  • flex-flow 是flex-direction属性和flex-wrap属性的简写形式
  • justify-content 设置子元素在主轴上的对齐方式
  • align-items 设置子元素在侧轴上的对齐方式
  • align-content 设置多行子元素在侧轴上的对齐方式(在单行情况下无效果)

子项常见属性

  • order 定义子项的排列顺序，数值越小，排列越靠前，默认为0
  • flex-grow 子项的放大比例。都为1时表示等分剩余空间，为0表示不放大
  • flex-shrink子项的缩小比例。如果所有都设为1，当空间不足时，都等比缩小；如果某个设为0，其余设为1，空间不足时，0不缩小
  • flex-basis 在分配多余空间之前，计算是否有多余空间，默认是auto，即项目本身大小
  • flex 是flex-grow, flex-shrink 和 flex-basis的简写，默认为0 1 auto
  • align-self 允许单个项目有与其他项目不一样的对齐方式，可覆盖align-items属性

应用场景

适用于移动端，小程序的布局，可方便实现元素的水平垂直居中，以及两栏三栏自适应布局

BFC

块级格式上下文。页面中的一块渲染区域，并且有一套属于自己的渲染规则。形成一个相对于外界独立的空间，让内部的子元素不会影响到外部的元素。

特点：

内部盒子会垂直方向一个接一个的放置

对于同一个BFC的两个相邻盒子，margin会发生重叠，与方向无关

每个元素的左外边距与包含块的左边界相接触

BFC的区域不会与float的元素区域重叠

计算BFC的高度时，浮动子元素也参与计算

BFC就是页面上的一个隔离的独立容器，内部子元素不会影响外部元素

形成条件：

浮动元素，float除none以外的值
绝对定位元素，position是absolute或fixed
overflow除了visible以外的值（hidden，auto，scroll）
display取值为其中之一（inline-block，table-cell，table-caption，flex）
body根元素

应用场景：

防止margin重叠

防止外部盒子因为内部元素浮动而自身高度为0

自适应多栏布局

伪类结构选择器

根据文档结构来选择元素，常用于根据父级选择器选择里面的子元素

选择符 简介
E：first-child 父元素中的第一个子元素
E：last-child 父元素中的最后一个子元素
E：nth-child(n) 父元素中的第n个子元素
E：first-of-type 指定类型E的第一个
E：last-of-type 指定类型E的最后一个
E：nth-of-type(n) 指定类型E的第n个

nth-child(n) ；n可以是数字(1,2,3,,,,)，关键字（even,odd），公式(2n,2n+1,5n,n+5,-n+5,,,,n从0开始递增取值)

nth-child(n)和nth-of-type(n)的区别：

nth-child(n)会先对所有子元素排序，找到第n个子元素（固定），然后看看是否和E匹配，而nth-of-type会先找到所有E，再排序选择第n个子元素
